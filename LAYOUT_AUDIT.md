# Mobile Layout Audit (~390px viewport)

Read-only audit. No files were edited. Scope: every file in `src/pages/` and `src/components/`.

Two bugs were given as confirmed reference cases:
1. `InvoiceDetailPage.tsx` — date inputs render right-aligned/overflow their box (fix already applied there).
2. `PayrollPage.tsx` — "Save all drafts" button clips off the header's right edge (not yet fixed).

This audit searched for the same two classes of bug (native-date-input misrendering, and non-wrapping flex rows with fixed-size children) everywhere else in the app, plus the other patterns requested (grids without breakpoints, unshrinkable buttons, content wider than parent).

## Patterns checked and found already handled correctly (no action needed)

- Card-header rows (`flex items-start justify-between`) in `ClaimCategoriesPage`, `LeavePage`, `TermsPage`, `RosterSettingsPage`, `PackagesPage`, `ClaimsPage`, `StudentsPage`, `ClassesPage`, `AttendanceConditionsPage`, `StaffDirectoryPage`, `StaffMemberDetailPage`, `KudosWallPage`, `StaffDocPanel` — all correctly pair a `min-w-0 flex-1`/`truncate` text block with a `shrink-0` badge/button.
- Form footer button rows (Save/Cancel, Update/Add, etc.) across `PackagesPage`, `ClassesPage`, `AttendanceConditionsPage`, `RosterSettingsPage`, `ClaimCategoriesPage`, `StudentsPage`, `WifiPage`, `ConfirmDialog`, `ReorderableTileGrid` — all use `flex-1` evenly on sibling buttons, no fixed/unshrinkable widths.
- `TabNav.tsx` — tabs use `flex-1` per item, no fixed widths.
- `AppHeader.tsx` — text labels are `hidden sm:inline`, so mobile shows icon-only controls; no overflow risk.
- `PayrollPage.tsx`/`OpeningBalancePage.tsx` wide tables — `overflow-x-auto` + `min-w-[…]` is the intentional horizontal-scroll pattern for tabular data, not a clipping bug.
- `ConfirmDialog.tsx` — backdrop has `p-4`, dialog is `w-full max-w-sm`; properly constrained at 390px.
- `grid-cols-2` occurrences without responsive breakpoints (`BoardPage`, `LeavePage`, `ClaimsPage`, `RosterSettingsPage`, `StaffMemberDetailPage`, `LeaveBalancesPage`, `RegisterStaffForm`, `StaffDocPanel`, `ReorderableTileGrid`) — checked individually; cell content (short labels, selects, single inputs) fits comfortably in a ~180px column at 390px. Not flagged.
